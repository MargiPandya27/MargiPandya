    Introduction to RAGs

    Large Language Models (LLMs), or any type of foundational models, are typically trained on static datasets. Retraining these models every time new data is added to the internet is not feasible. While fine-tuning a model for a specific domain may be effective in certain use cases, it is not a scalable solution for keeping up with daily information updates.

    For example, if you ask ChatGPT to provide papers on "Multi-Agent Systems for Autonomous Driving," it may respond with a curated list of papers up to 2023 or 2024. This is useful—but what if you want to retrieve papers published just yesterday? In fields like Machine Learning and Deep Learning, hundreds (if not thousands) of papers are published every day. Constantly retraining the model is simply not practical.

    This is where Retrieval-Augmented Generation (RAG), Prompt Engineering, and Fine-Tuning come into play. These techniques help ground the responses of foundational models based on the specific query being asked. Each approach has its pros and cons, and the best choice depends on the application.

    What is RAG?

    RAG can be thought of as a data pre-processing layer for LLMs. Instead of feeding the model a massive corpus of documents (webpages, PDFs, etc.), only the most relevant document chunks are provided as context along with the user's query. The LLM then generates a response based on this grounded context.

    At a high level, RAG involves three key components:

    • Indexing
    • Retrieval
    • Generation

    1. Indexing

    Indexing involves breaking documents into smaller chunks that fit within the context window of an embedding model. These chunks are then passed through an embedding model to convert them into vector representations. These vectors capture the semantic meaning of the text and place similar concepts close together in vector space. The resulting vectors are stored in a vector database.

    2. Retrieval

    When a user submits a query, it is also converted into a vector using the same embedding model. The system then searches for the most semantically similar chunks in the vector database.

    A simple method for retrieval is k-Nearest Neighbors (k-NN), which calculates cosine similarity between the query vector and vectors in the database. However, this method has a time complexity of O(N), which becomes inefficient as the dataset grows.

    To optimize this process, Approximate Nearest Neighbor (ANN) search algorithms are used. One popular method is Hierarchical Navigable Small World (HNSW) graphs, which use skip-linked lists and graph structures to significantly reduce the search time to O(log(N)).

    3. Generation

    This stage involves two main components:

    • Prompt Engineering: Techniques like zero-shot learning, few-shot learning, and chain-of-thought reasoning are used to guide the model output effectively.
    • Choice of LLM: Based on token limits, speed, accuracy, cost, and integration feasibility.

    All-in-One RAG Pipeline (LangChain + HuggingFace)

      
  
# 1. Load documents
from langchain_community.document_loaders import TextLoader, PyPDFLoader

# Load from .txt
txt_loader = TextLoader("sample.txt")
txt_docs = txt_loader.load()

# Load from PDF
pdf_loader = PyPDFLoader("/content/Survey_Amodal_completion.pdf")
pdf_docs = pdf_loader.load()

# Combine all documents
docs = txt_docs + pdf_docs

# 2. Chunk the text
from langchain.text_splitter import RecursiveCharacterTextSplitter

raw_texts = [doc.page_content for doc in docs]

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=100
)
texts = text_splitter.create_documents(raw_texts)

# 3. Generate embeddings
from langchain.embeddings import HuggingFaceEmbeddings

embedding = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")

# 4. Create vector store
from langchain_community.vectorstores import FAISS

db = FAISS.from_documents(texts, embedding)
retriever = db.as_retriever()

# 5. Retrieve relevant documents and prepare prompt
query = "What are the related works that have been done on this?"
retrieved_docs = retriever.get_relevant_documents(query)
context = "\n\n".join([doc.page_content for doc in retrieved_docs])

prompt = f"""
You are reading a research paper. The following is extracted content from the abstract and related work sections:

{context}

Please summarize the related work section of the paper, highlighting prior approaches, key methodologies, and how this paper builds upon or differs from them.
"""
