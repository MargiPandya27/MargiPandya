    RAGs for Research Papers

    Overview

    This project implements an advanced Retrieval-Augmented Generation (RAG) system tailored for academic research papers. It allows users to query either a predefined corpus or upload their own papers for real-time processing, embedding, and retrieval. The system supports two modes:

    • Predefined Corpus Mode: Operates on a curated dataset of academic papers.
    • User-Provided Papers Mode: Enables users to upload their own documents for on-the-fly processing.

    Key components of this system include:

    • Document preprocessing
    • Semantic chunking
    • Embedding generation using SciBERT
    • Efficient similarity search using FAISS with HNSW indexing
    • Semantic routing to distinguish between content-based and metadata-based queries

    Plan

    The goal is to enable the system to respond accurately to user queries such as:

    “From the given context, can you explain this concept?”
    “Which papers published after 2024 discuss X?”

    Embedding Model

    At the core of the pipeline is an embedding model trained on scientific literature—SciBERT—used to generate semantic vector representations of text. To perform efficient similarity search, we use FAISS (Facebook AI Similarity Search) in combination with Hierarchical Navigable Small World (HNSW) graphs. HNSW enables fast and scalable approximate nearest neighbor searches in the vector space.

    Semantic Chunking

    When a user uploads a PDF, the system performs semantic chunking instead of random text segmentation. It breaks the paper into meaningful sections like Abstract, Introduction, Related Work, and Methodology. This approach preserves contextual integrity and improves retrieval performance by aligning chunks with the paper's logical structure.

    Semantic Routing

    Semantic routing helps direct user queries to the appropriate processing logic. The system distinguishes between:

    • Content-based queries (e.g., “What methods are used for X?”)
    • Metadata-based queries (e.g., “Show papers published after 2024 on Y”)

    This enables flexible querying of both the full text and metadata without overloading a single retrieval mechanism.

    Evaluation Metrics for RAG

    • Retrieval Accuracy: How frequently relevant documents are returned.
    • Response Quality: Evaluation (human or automated) of the generated answers.
    • Latency: Time taken to respond after a query is issued.
    • Scalability: System performance as the corpus size increases.
    • Metadata Precision: Accuracy in extracting and returning structured information (e.g., author names, publication years).
