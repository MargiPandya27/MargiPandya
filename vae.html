Variational Autoencoder

Variational Autoencoder (VAE) is a kind of autoencoder that was originally used to generate images, but by changing the decoder, we can make it generate text too.

Main Motivation

Given a set of training data of faces, the neural network can be trained to understand the distribution of the semantic/hidden features of the images provided, and the same set of hidden features can be sampled to generate similar images.

For the training data \( X \), our goal while training is to maximize the likelihood of the data \( p(X) \). From Bayes Theorem, \( p(X) \) can be written as the summation of the joint probability distribution of the given data and the hidden semantic features:

\[ \max p(X) = \sum_{z \in Z} p(x, z) = |Z| \cdot \frac{1}{|Z|} \sum p(x, z) = \mathbb{E}_{z \in Z}[p(x, z)] \]

It is intractable to sum over all values of \( z \). If \( z \) is continuous, then \( p(x) \) becomes an integration, which is computationally intractable. Thus, we need a sampling technique to consider only the values of \( z \) that significantly contribute to \( p(x, z) \).

Sampling Technique

1. Monte Carlo Sampling

For most values of \( z \), \( p(x, z) \) is low and not useful for training. Instead:

\[ p(x) \approx |Z| \cdot \frac{1}{k} \sum_{i=1}^{k} p(x, z^{(i)}) \]

2. Importance Sampling

Involves sampling from another distribution \( q(z) \):

\[ p(x) = \sum_{z \in Z} \frac{q(z)}{q(z)} p(x, z) = \mathbb{E}_{z \sim q(z)}\left[\frac{p(x, z)}{q(z)}\right] \]

Sample \( z^{(1)}, z^{(2)}, ..., z^{(k)} \sim q(z) \):

\[ p(x) \approx \frac{1}{k} \sum_{i=1}^{k} \frac{p(x, z^{(i)})}{q(z^{(i)})} \]

Likelihood Estimator

The likelihood of data \( X \) parameterized by \( \theta \) is:

\[ p(X; \theta) = \sum_{z} p(x, z) \]

For i.i.d. data:

\[ p(X) = p(x^{(1)}) \cdot p(x^{(2)}) \cdots p(x^{(m)}) \]

Taking log-likelihood:

\[ \log p(X; \theta) = \log \left( \prod_{i=1}^{m} p(x^{(i)}) \right) = \sum_{i=1}^{m} \log p(x^{(i)}) \]

Now consider:

\[ \log p(x; \theta) = \log \left( \sum_{z} q(z) \cdot \frac{p(x, z)}{q(z)} \right) \]

Using Jensen's Inequality

For a concave function \( f \), Jensen's inequality states:

\[ f(\mathbb{E}[x]) \geq \mathbb{E}[f(x)] \]

Applying to the log-likelihood:

\[ \log \left( \mathbb{E}_{z \sim q(z)} \left[ \frac{p(x, z)}{q(z)} \right] \right) \geq \mathbb{E}_{z \sim q(z)} \left[ \log \left( \frac{p(x, z)}{q(z)} \right) \right] \]

The right-hand side is the **Evidence Lower Bound (ELBO)**.

Condition for Equality

Equality in Jensen’s inequality holds when \( q(z) = p(z|x) \). Then:

\[ \sum_z q(z) \log \left( \frac{p(x, z)}{q(z)} \right) = \sum_z q(z) \log \left( \frac{p(z|x)p(x)}{q(z)} \right) \] \[ = \sum_z q(z) \log \left( \frac{q(z)p(x)}{q(z)} \right) = \sum_z q(z) \log p(x) = \log p(x) \]

Approximating the Posterior

For a tight bound, we assume \( q(z) = p(z|x) \). But in practice, we approximate the posterior using a neural network:

\[ q_{\phi}(z|x) \]

If we assume a standard normal prior: \[ p(z) = \mathcal{N}(0, I) \] then we aim to learn the distribution \( p(z|x) \) using a neural network that outputs the mean and standard deviation.

Generation Process

To generate data, we:

  1. Sample \( z \sim q_{\phi}(z|x) \)
  2. Use the decoder to get \( x \sim p_{\theta}(x|z) \)
The joint distribution is:

\[ p(x, z) = p(x|z) \cdot p(z) \]
